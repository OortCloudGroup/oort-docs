# H5轻应用

# 定义

H5是指第5代HTML，也指用H5语言制作的一切数字产品。 所谓HTML是“超文本标记语言”的英文缩写。“超文本”是指页面内可以包含图片、链接，甚至音乐、程序等非文字元素，而“标记”指的是这些超文本必须由包含属性的开头与结尾标志来标记。我们上网所看到的网页，多数都是由HTML写成。它是一种高级网页开发技术，他不是一种标记语言，为下一代web提供了全新的框架和平台，包括提供免插件的音视频、图像动画、本体存储以及更多重要的功能。相比H4,H5有更多的交互和功能，并且支持在移动设备上支持多媒体。

# 应用

H5的应用很广泛，基于H5有很多产品载体，比如PPT、APP等，但这些产品载体在实现过过程中，也会用到其他的技术。 而从营销层面，最常用的是H5页面，这些页面可以单独传播，即H5海报，也可以嵌入公众号，嵌入APP、嵌入小程序等，这种形式并不仅仅可以在微信端显示，作为一个页面，获取链接后，也可以在PC端打开。 目前主要的形式包括：

  • HTML5的游戏开发
  • 轻应用、Web App、微站，网站包括PC端和移动端，响应式网站适配不同的终端
  • Hybrid App
  • 基于微信平台的开发
  • Web VR让虚拟现实大众化
  • 动漫、二次元

# 移动端原生应用

  • 原生开发（Native App开发），是在Android、IOS等移动平台上利用官方提供的开发语言、开发类库、开发工具进行App开发。比如Android是利用Java、Eclipse、Android studio；IOS是利用Objective-C 和Xcode进行开发。

# 优点：

  1. 可访问手机所有功能（如GPS、摄像头等）、可实现功能最齐全；
  2. 运行速度快、性能高，绝佳的用户体验；
  3. 支持大量图形和动画，不卡顿，反应快；
  4. 兼容性高，每个代码都经过程序员精心设计，一般不会出现闪退的情况，还能防止病毒和漏洞的出现；
  5. 比较快捷地使用设备端提供的接口，处理速度上有优势。

# 缺点：

  1. 开发时间长，快则3个月左右完成，慢则五个月左右；
  2. 制作费用高昂，成本较高；
  3. 可移植性比较差，一款原生的App，Android和IOS都要各自开发，同样的逻辑、界面要写两套；
  4. 内容限制（App Store限制）；
  5. 必须等下载完毕用户才可以打开，获得新版本时需重新下载应用更新。
  6. 新需求迭代，上线慢。

# 创建应用

  • 2，创建应用

# 1.登录平台

  • 通过登录界面登录供应商开发平台。

    进入网站首页、并点击“我的应用”就可以创建应用

# 2.创建应用

以创建一个名为demo的H5应用为例。

  • 2.1 点击 【+ 创建应用】，进入创建应用对话框。

    • 应用类型：移动端原生(android)、移动端原生(ios)、移动端H5、PCWEB、PC桌面端
    • 应用名称：最大长度为8个字
    • 应用图标：应用的logo
    • 应用终端类型：移动端、大屏(华为IdeaHub、ipad)等、兼容
    • 应用分类：科研助手、学生管理、移动办公、队伍建设、沟通协作、AI+视频、基础工作
    • 一句话介绍：最大长度为256个字

  • 2.2 在弹出框里填写应用的类型、名称、应用图标、分类、以及应用的简单概要信息。

    点击【创建】按钮后，初步完成应用的创建。此时，首页即可出现新创建的名为demo的应用了。

    注意此时应用的状态为【开发】状态,此状态代表应用还没有全部完成，我们需要进一步编辑应用信息。

  • 2.3 编辑应用

    • 2.3.1 点击应用右上角的【...】在下拉中选中编辑应选项

    进入编辑界面。

    • 首先我们可以看到应用的名称、应用的分类是前面创建好的。另外我们看到在【应用凭证】 多了应用ID、应用密钥两个属性。应用ID是应用的标识、应用密钥是为了能够获取token 这两者都是在创建应用时后台自动生成的。如果想修改应用的名称、分类等，可以点击应用图标右侧的【🖊】。

    • 2.3.2 编辑应用凭证和基本信息

    上传应用包、应用截图、指定应用编码、应用程序入口、版本号（默认为1.0.0版本）、功能简介。

    注意这里的上传的应用包格式，需要解压后的文件夹就是项目的主目录。

    • 2.3.3 编辑服务信息

    服务信息主要有3大项：

    • 综合信息：
      • 是否强制更新，默认为否
      • 应用部署网络：默认一类网
    • 建设信息：应用归属地、应用开发时间、应用责任人、建设单位、联系电话
    • 支持信息：应用开发单位、运维电话

    • 2.3.4 应用发布管理

      • 使用范围：全局、无、部分可见
      • 发布时间：立即发布、指定时间
      • 版本描述：当前版本的简单描述

      底部可以查看版本的历史记录。编辑完这些信息，点击【发布】。

    此时在首页中应用的状态由【开发】变为【待审核】。

    至此，完成应用的创建，待管理员审核通过后，应用的状态就会由【待审核】变为【已通过】，这样就发布成功， 在应用适用范围内的所有人都可以时应这个应用了。
